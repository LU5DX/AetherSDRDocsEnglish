# Understand how to contribute bug reports and PRs

This page explains how to open the bundled contributing guide in AetherSDR. The guide covers how to file bug reports and submit pull requests for the project.

## Before you start

- No radio connection is required to open help topics.

## Steps

1. Click `Help` in the menu bar.
2. Click `Contributing to AetherSDR...`.
3. The HelpDialog opens, displaying the contributing guide in the Markdown viewer.
4. Read the content. External links in the document open in your default browser.
5. Click `Close` when finished.

## What each control does

| Control | Description |
|---|---|
| `AETHERSDR OFFLINE HELP` | Brand header shown above the topic title. |
| Title | Displays the name of the open help topic. |
| Subtitle | Shows a one-line description: "Bundled help is available even when your station computer is offline." |
| Markdown viewer | Renders the contributing guide. Links are clickable and open externally. |
| Hint / footer | Displays: "Tip: The Help menu keeps each guide separate so you can reopen just the topic you need." |
| `Close` | Closes the dialog. |

## Tips

- Each help topic in the `Help` menu opens its own separate dialog, so you can reopen just this topic without navigating through other guides.
- If the contributing guide fails to load, AetherSDR displays a fallback message in the Markdown viewer indicating the help file is not available. In that case, reinstall the application or report the missing help asset.
